# Global Tech Stack

## When to use this skill

- When choosing technologies or frameworks for new features
- When adding new dependencies or libraries to the project
- When setting up project infrastructure and build tools
- When configuring package managers (npm, yarn, pnpm, pip, bundler, etc.)
- When integrating third-party services (authentication, email, monitoring, etc.)
- When selecting appropriate frameworks, ORMs, or UI libraries
- When ensuring consistency with existing technology choices
- When documenting technology decisions and rationale
- When reviewing compatibility of new tools with existing stack
- When onboarding team members to the project's tech stack
- When updating or upgrading major dependencies
- When considering alternative technologies or refactoring decisions

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global tech stack.
